## **前言**

“人要怎么度过一生？”，无聊的时候我有时候会想这个问题，这个问题太大了，我一直没有答案。前几个月，有一次聊天后，在想“人为什么要工作？为什么有时候不开心也要上班？”的时候，突然有一些思路了。

可能大家会觉得好笑，但首先，肯定得活着。作为一个生物体，人活着，在生理层面上，需要氧气、需要保持恒温、需要摄入营养素、需要新陈代谢、需要休息、需要避免物理伤害、需要抵御外部病原体侵害。呼吸、新陈代谢和免疫系统运作是人无意识即可进行的活动，而保持恒温、摄入营养素、休息、抵御外部威胁需要一些有意识的活动。

- 保持恒温，可以通过衣物、燃料（柴火、煤炭等）和现代化设备（空调、暖气等）等解决严寒；可以通过扇子、冰块和现代化设备（电扇、空调等）等解决酷热。
- 摄入营养素，需要进食。
- 休息、避免物理伤害，需要有一个安全的环境，比如，有巢氏创造了房屋。
- 抵御外部病原体侵害，需要有一个相对干净、卫生的环境，可能还需要药物、治疗等。

衣物、燃料、食物、住所、药物等都可以概括为物资。

获取物资的途径有：大自然的馈赠、生产、交换等。大自然的馈赠可能需要加工生产才能使用，而且不是所有人都能生产所需的所有物资，所以需要交换。交换应该是最常见的获取物资的途径。交换可以以物换物，但不太便利，所以需要一个可以衡量物资价值的中间工具——货币。

问题来到，如何获取货币？

- 可以通过他人赠予获取。比如，祖辈积累等。
- 可以通过交易获取。交易的前提是有可以交易的东西，可以是大自然的馈赠，可以通过生产获取，也可以是一些无形的东西，也可以低价买入高价卖出赚取差价。
- 也可以通过劳动获取。劳动，可以理解为，帮有货币的人做事，由他们分配货币，然后劳动者再通过货币去交换，获取物资。这个可以简称为工作，大部分人是通过工作谋生的。（或许工作也可以理解为交易的一种，只是劳动者交易的是时间、精力和技能）
- ……

还有个问题，需要多少？如果是有劳动能力和自理能力的人，其实很容易获取生存所需物资。但人不是一生下来就是成年人，而且也不是一直都有劳动能力和自理能力的。幼儿、无法自理的老年人等无法获取货币或获取得不够的人能活着，是因为有人帮助他们。大部分应该是家庭成员的帮助，小部分是社会资源再分配等。因此，一个人需要获取能保证家庭成员（可能包括儿女、自己、父母、伴侣等）生存所需的资源。

当保障了生存后，才有时间想怎么度过剩下的时间，即在剩下的时间里做什么。也可以这么理解，人生要做很多事，可以分为两类，一类是生存之内必做的事，另一类是生存之外选做的事。

不管是交易、劳动，还是生存之外想做的事，至少都需要思考两个问题：做什么事情、怎么做。这两个问题没有明确答案时，可能都会迷茫。

> 题外话：
>
> 可能有人会想“人为什么要活着？”。当想到这个问题时，已经是一个有意识的存在了，不需要想“为什么要存在”，不需要理由。应该去想，“我要如何存在？”。
>
> 也可能有人会想“我是怎样的存在？”，也没有必要，应该去想，“我想成为怎样的存在？”。



## 做什么事？

[八木仁平《如何找到想做的事》](https://weread.qq.com/web/reader/71a32fb0813ab8de8g019cc9#outline?noScroll=1)中说，真正想做的事是重要的事（价值观）、擅长的事（天赋）、喜欢的事（热情）的交集。要用天赋，在感兴趣的领域去做重要的事情。重要的事不会让我们厌倦；擅长的事会让我们相信自己能做到、做好；喜欢的事会让我们发挥主观能动性。

> 书中这样解读价值观：一类是面向自己的，个人觉得重要的事（人生目的），比如，权利、名声、挑战、家庭、事业等；另一个类是面向他人和社会的，个人想要传播的影响（工作目的^[1]^），比如，自由、创造、科技等。
>
> [^1]: 原文不确定是不是“工作目的”，个人觉得用“事业目的”概括更准确些。与工作区分开，是不计回报也想去做的。

工作与想做的事没有关系。工作中的事可能是想做的，也可能不是想做的。

受这个思路启发，可以按照下面表格的思路去选择：（数字表示优先级，数字越小优先级越高）（每个人的排序可能不一样）

|                          | 工作、交易 | 想做的事 |
| ------------------------ | ---------- | -------- |
| 重要的事（价值观）       | 2          | 1        |
| 擅长的事（天赋/技能）    | 1          | 3        |
| 喜欢的事（感兴趣的领域） | 3          | 2        |

工作是一个双向匹配的过程。招聘方有一些事情需要人做，应聘方有一些技能可以做一些事情。对于应聘方来说，首先要有一些技能保证满足招聘方的需求，然后才是考虑事情的目标、做事的过程与个人价值观的匹配度。虽然工作要求的是技能，但天赋可能是职业生涯的分水岭。能做和做得好是不一样的水平（收获的货币和有价物也是不一样的）。

如果是通过交易获取货币，首先要考虑有哪些需求，评估回报率、周转率、增长空间、风险等，是否值得做，然后考虑己方是否擅长解决这些需求，评估竞争力等。和工作类似，对于己方来说，首先要考虑的也是擅长的事，优劣势。

如果按照《如何找到想做的事》中所说，从三类事情的交集中找真正想做的事，可能有点复杂，需要很长时间。个人觉得，以自己舒服的状态做喜欢做的事情开始，慢慢去找就好了。

- 难受的点，反过来，肯定是重要的事。
- 如果特别喜欢，就算失败多次也会坚持，其实也不用考虑有没有天赋，做就好了。
- 只是大部分人，可能会因为多次失败而迷茫，或者不确定能不能做到、做好。所以，最好还是擅长的事，更能有成就感，更容易坚持，更容易成功。

工作中的事是自己想做的事，意味着，以自己舒服的状态做喜欢做的事情，是自己想要传播的影响，并且满足了生存所需，这是非常幸运的。

上面只是提供了选择的思路，那怎么知道：

  - 可以通过互联网搜集信息
  - 可以看书、旅行、观察，了解这个世界发生了什么事、其他人在做什么……
    - 分享最近看到的一段文字：“我们在一个个场所、机构和文化中度过了一生——有些是在现实世界里，有些则存在于虚拟网络中——这些场所、机构和文化强化了我们原有的世界观，却并未将其拓宽。至于世界上的其他人都在信仰什么、在做些什么，我们却从未真正接触过。世界比我们所知的更丰富、更精彩。<u>从这个层面上来说，旅行与享受无关。作为一种接触体验的形式，旅行成为最重要的学习形式，因为它是鲜活知识的主要来源</u>。”

可能遇到的一些问题：

- 做的事情的目标可能违背自己的价值观，事情的目标是个人无法改变的，又不能放弃工作，比较痛苦，怎么办？
  - 首先要思考问题是否存在，做的事情的目标是否真的违背自己的价值观。如果是听到一些负面的声音，那是否有数据衡量占比、影响、真实性。别轻易下结论。如果问题很重要，充分调研、了解、思考再下结论。
  - 如果真的很痛苦，其实换一件事情做，换一份工作也没有那么难。可能回报会少一些，但开心一些。
- 做事的过程（包括但不限于任务分配、沟通协作等与人打交道的事情）让人难受，怎么办？
  - 相比客观问题，主观问题可能更难处理。具体事务是客观的，而与人打交道则充满不确定性。比如，分配不公、沟通不当、协作混乱、价值观或风格冲突等。
  - 哭泣、发泄情绪都是正常的，但总要解决问题。
  - 可以先坦诚沟通，表达自己的述求，倾听对方的想法。可能对方也是很好的人，只是刚开始不熟悉。可以先给自己和对方机会。
  - 达不成共识，问题无法解决，又很影响自己的话，其实也可以换一个团队，换不了可以想想其他方法。


一些想法：

- <u>不管怎样，选择自己觉得更重要的就行</u>。只要所处的环境能种出粮食，自身有自理能力，饿死的概率很小。做不了脑力劳动，可以做体力劳动，找不到工作可以去种地，不会种地可以学，没有地就租或者开荒。
- 理论上，越早知道自己想做的事，可以投入的时间越长，越容易成功。但<u>什么时候都不晚</u>，即使可能收获不了很高的成就，但对于个人来说，这也是人生意义上的一种成功。
- <u>不管想做的事是什么，自己觉得好就行</u>。有些人的梦想可能是结婚生子，有些人的梦想可能是事业成功。前者可能向往温馨的家庭生活，后者可能追求的是突破与创造。不管是哪一种，都没有高低贵贱之分。事本无义，义由心生。
  - 最近有感：理发师也是一个很好的职业。对于理发师来说，美可能是重要的事情，并且是重要到想分享给其他人的。人们可能因为追求“美”去理发店，理发师帮助人们追求“美”。<u>如果过程是双向奔赴的，就会觉得很浪漫</u>。不仅是理发师，健身教练、厨师、农民、演员、科技工作者等其他职业也是。职业没有高低贵贱之分，自己享受在其中就好。

- <u>在找重要的事情时，建议追溯到根源</u>。避免“成功”后的迷茫。避免在追逐“假的”重要的事情时丢掉真正重要的事。
  - 比如，有些人可能觉得自己想做的事是赚很多很多钱，这样的话可以再想想，想用钱交换什么？有了这些钱后想做什么？钱最终是通过交换才有价值。
  - 比如，不同的人喜欢骑行的原因可能是不一样的，有些人喜欢被风吹佛过的、鲜活地活着的感觉，有些人觉得可以有自己的时间，有些人觉得可以放空自己，有些人喜欢欣赏风景，有些人觉得可以锻炼身体，有些人觉得可以减肥……追溯到根源后，会发现实现目的的方式不止一种，而且会更了解自己。



## 怎么做？

明确了“做什么”后，更关键的问题是“怎么做”。最好是有一套通用的行动框架。每个人都可以构建自己的框架，并不断更新。以下是个人框架的更新历程。

在[极客时间的专栏《10x程序员工作法》](https://time.geekbang.org/column/intro/100022301?utm_campaign=geektime_search&utm_content=geektime_search&utm_medium=geektime_search&utm_source=geektime_search&utm_term=geektime_search)中，介绍了一种适用于程序员的思考框架：

- 以终为始
- 任务分解
- 沟通反馈
- ~~自动化~~ -> 工具化

前三项很好理解，第四项用“工具化”代替或许更合理。

以终为始解决方向和距离问题；任务分解处理复杂度问题；沟通反馈解决人与人之间的问题；而工具化，则是借助外物——无论是软件、硬件、实物、清单，还是人工或自动流程——来辅助达成目标，以节省人力、提升效率、确保正确性，甚至完成人力所不能及的事。工具化的发展历程，比如农具从石耜（石器，人力）到铁犁（铁器，畜力），再到拖拉机（机器，动力）乃至智能农机（自动化设备，数字）的演进。即使无法一步实现自动化，也可以从简单工具起步。

后来又想了下，沟通反馈有点片面，或许用“团队协作”更完整。除了沟通反馈，团队运作还需要制度与规范。

- 以终为始
- 任务分解
- 团队协作
  - 沟通反馈
  - 制度规范
- 工具化

再进一步思考，还缺少关于“方法”的维度。因此补充“方法化”。几乎所有的事情都有方法，决策、学习、沟通、种地、营销等。农耕从刀耕火种到精耕细作，排序算法从冒泡排序到快速排序，业务流程的步骤优化，零售陈列中的关联策略（比如“啤酒与纸尿裤”案例），都是方法的演进与应用。

- 以终为始
- 任务分解
- 团队协作
  - 沟通反馈
  - 制度规范
- 工具化
- 方法化

后来在一次聊天中，聊到“要事第一”，联想到《高效能人士的七个习惯》，发现还少了自我管理和优先级管理，于是加入“积极主动”“要事第一”。并且将“团队协作”扩展为更为通用的“人际协作”。知彼解己、双赢思维、综合综效可视为沟通反馈的具体方法。

在整理的过程中，发现还没有闭环，少了复盘。完整的流程应该是这样的：实践 → 复盘 → 发现问题与规律 → 驱动方法化（优化策略）→ 驱动工具化（引入或改进工具）→ 再次实践 → 再次复盘。复盘是对“终”的再确认和对“始”的再校准，是“任务分解”与“要事第一”的校准器，是协同过程的优化器，是“工具化”和“方法化”的源泉。

以下是目前的最终版本（不够通用全面，仍会持续迭代）：

| 阶段     | 核心习惯/原则 | 说明                                               |
| -------- | ------------- | -------------------------------------------------- |
| 内核驱动 | **积极主动**  | 关注可影响的事情，存在问题就解决问题，立即执行     |
| 导航规划 | **以终为始**  | 明确方向与距离，确保行动始终对准结果               |
|          | **任务分解**  | 拆解复杂性，将大问题转化为可执行的步骤             |
|          | **要事第一**  | 建立优先级意识，主动规划，避免陷入琐事循环         |
| 人际协作 | **沟通反馈**  | **知彼解己、双赢思维、综合综效**                   |
|          | **制度规范**  | 提供协作的结构与依据                               |
| 赋能进化 | **工具化**    | 善用外在手段节省人力、提升效率、保证质量，达成目的 |
|          | **方法化**    | 通过优化策略、流程与算法，实现系统性进步           |
|          | **复盘**      | 回顾过程、提炼规律、迭代系统，驱动持续进化。       |



### 一些思考

> 胡说八道。

**怎么做业务？（研发角度）**

在业务部门，其实只有一件事，就是干好业务。业务干不好，技术团队也存活不下来。在业务部门里面做技术，其实就是用技术手段（软件或者硬件技术等）实现业务工具。

做一件事情基本都有这些步骤：以终为始、任务拆解、协作（沟通反馈、团队管理、进度管理等）、工具化、方法化、复盘。不管是做整个业务，还是小到一个需求，都是这些步骤。

- 业务或产品同学确定“终”，定义目标，怎么才算完成目标，具体表现可能是定性的描述、定量的数据。
- 业务或产品同学根据目标拆解任务，确定任务优先级，具体表现可能是 MRD、BRD、PRD。
- 产研同学协作完成任务。

按照要求完成工具，保证工具的稳定和性能，其实只是基本要求。更高的要求应该是为业务带来新的增长点。

怎么用技术促进业务发展，感觉就是站在用户的角度去想他们缺少什么工具、业务步骤等方法是否可优化。这个时候，要忘记你使用的技术，不局限于你能提供的，而是想对方缺少的、需要的。比如，软件技术人员也可以想是否可以用硬件完成目的。甚至忘记你是技术人员，去想操作动作、顺序、等待时间、策略等是不是有改进空间。



**怎么让自己成长？怎么帮助他人成长？**

> 歪个话题，也想过为什么有些人会想离职或换团队？
>
> - 有可能是事情目标与价值观冲突
>   - 业务大方向的终、业务形态等是技术无法动摇的，技术同学是做工具的人，没有决策权。如果这个“终”与个人价值观冲突，就会很痛苦。
> - 有可能是协作痛苦
>   - 事情基本是以团队的方式完成。沟通反馈解决人与人之间的问题。为什么人跟人之间会有问题？因为每个人的性格、价值观、信任度、内核状态、对事情的看法等可能都会有差异。
> - 有可能是无需担忧生存问题并且个人重要的事得不到满足，比如，
>   - 可能事情简单得不到锻炼，满足不了成长诉求；
>   - 可能休息时间不够或影响身体健康；
>   - 可能没有自由；
>   - 可能兼顾不了家庭；
>   - 可能收入不达预期等等
> - 有可能不是自己喜欢的事且有底气追求喜欢的事
> - ……

成长也是有步骤的。

- 知道缺失的东西。成长意味着现状和目标存在距离。
  - 可以通过外界发现。比如，老师、导师的提醒。
  - 可以通过自省发现。
- 知道怎么补充缺失的东西。“和高人谈，从书上学，在事上练”这句话总结得很好。
  - 外部补充
    - **和高人谈**，了解、观察高人做事的方法，比如，专家、资深人士
    - 接触不到高人，就**从书上学**
  - **在事上练**，需要亲身实践，将外部补充转化为自己的东西
  - **刻意练习**
  - 最后，总结出自己做事的方法并不断优化，沉淀知识，积累经验，迭代认知



**关于 AI 的一些想法**

软件、硬件是工具。AI 既可以是工具，也可以是制作工具的工具。AI 不仅推动<u>工具的自动化生产</u>，还推动了<u>思考与创造的自动化</u>。

我们需要适应的，是从“如何做”的细节中解放出来，更专注于 “做什么”、“为何做”以及“做到何种标准” 这些更根本的问题。



### 一些方法分享

**如何写好文档？**

- 其实自我感觉写得一般。但也确实收到了一些认可，给大家留下的印象之一可能也是文档写得好。回想了下，个人觉得写得比较有意思的文档，2022 年 JavaScript 分享，2024 年 dt 微前端分享、xx 业务概述，2025 年 B 业务文档。
  - JavaScript 分享写得一般。但这是记忆中第一次比较正式的分享，写得时候也努力去想，要怎么把事情讲清楚，结构应该是怎么样的。虽然是参考模仿《重学前端》写的，但也有一点点结构的雏形了。
  - 微前端分享，完全没有在意格式，可能有点放飞自我，但把事情讲清楚了。
  - xx 业务概述，个人满意的点是，完全是自己的想法。微前端分享时，其实是分享现有的东西，难点是怎么讲清楚，但 xx 业务概述中有了自己的思考。而且，全文思路体感上只用了几分钟或者十来分钟，画图用得时间更多。
    - 当时的写作背景是，换到新业务 B，在了解新业务 B 时也想给接手之前的业务 A 的人介绍一下，虽然接触业务 A 的时间也不长。
    - 不管是什么业务，第一个问题是，为什么要做这个业务？
    - 在思考业务 A 为什么要存在时，觉得要把整个部门业务都讲一下，才能回答问题。所以，就重写了几年前写的 xx 业务概述。
  - B 业务文档，有了更多的思考，开始觉得做业务有意思了。
- 回归正题，或许可以复用的经验：
  - 先把问题想清楚。其实就是多问一些为什么、是什么、怎么做之类的问题，问到完全没问题，逻辑上也闭环，就可以了。
    - 比如，问题是什么？这是问题吗？为什么这是个问题？问题的原因是什么？要怎么解决？这些解决方式会有副作用吗？还有哪些方法？怎么知道是否解决了？……
  - 以幼儿园的小孩也能理解的方式去讲。
    - 比如，如果你的孩子问你，你的工作是什么？你要怎么讲清楚。注意，小孩的问题可能会一个接着一个。



**如何控制情绪波动振幅？**

常说的“情绪稳定”只是一种表象。有可能是刻意压制下的平静，内在情绪被掩盖，不知何时会爆发，这其实是一种不稳定的“稳定”；有可能是一种近乎无波的“水平线稳定”，但这过于理想。更合理的状态是将情绪的波动控制在合理、健康的振幅之内。

为什么要控制情绪？保持冷静，理性思考，可以避免错误决策，可以更好地实现目标、解决问题，并建立有效协作。而且负面情绪对身体不好。

怎么控制情绪？其实不是控制，是选择。情绪其实是自己选择的。

怎么能让自己不选择负面情绪？负面情绪太多了，仅分享最近关于“愤怒”的认知迭代。

- 如何避免愤怒？
  - 首先，判断是否重要。
  - 如果是不重要的人，不重要的事，完全不用理会。
    - 但可能还是会觉得膈应。最近领悟到的一个方法是，关注点偏离。想想自己重要的事，就会忘掉不重要的事。比如，看看别人那么优秀，你还有一大段距离，就没有心思关注路边的声音，只会想往前赶路。
  - 如果是影响到切身利益的事，解决问题就好了，不用想“为什么别人没有按照你预期的去做”。
- 其他认知迭代
  - 积极主动，有问题就解决问题。
    - 焦虑：想要的话，就去努力。不适合的话，就放弃。坚定目标之后，勇敢的去做就行。
    - 恐惧：其实没有什么大不了。

无法选择正面情绪，怎么调节负面情绪？向别人诉说，可能有一些用。但最终，其实还是需要自己去调节。调节对事物的看法。心理咨询最终也是引导咨询者自己解决问题。 

- 大自然效应
- 运动
- ……

其实，当到一个阶段后，可以更客观的看待一切事物，可以包容很多事情，自然就可以选择正面情绪。但这种状态需要自己去领悟，外界无法帮助。外界或许可以提供建议、指导，但最终决定自己的路的只有自己。

> 朋友，去问河水吧！你听，它在发笑！你果真相信，你的蠢行，能免除他的蠢行？难道你通过教育、祈祷和劝诫，能保他免于轮回？亲爱的，你曾对我讲过引人深思的婆罗门之子悉达多的故事，难道你完全忘记了？是谁保护沙门悉达多免于罪孽、贪婪和愚昧？是他父亲的虔诚，老师的规劝，还是他自己的学识和求索？人独自行过生命，蒙受玷污，承担罪过，痛饮苦酒，寻觅出路。难道有人曾被父亲或老师一路庇护？亲爱的，你相信有人能避开这道路？或许小悉达多能，因为你爱他，你愿意保他免于苦难和失望？但是就算你替他舍命十次，恐怕也不能扭转他命运的一丝一毫！

> ——《悉达多》

个人惊艳过的作品：

  - 写得非常好。看的过程中很多次被触动到想哭。看完或许会更理解为什么父母有时候会说，“你成家了，我就完成任务了”“这是我的任务”。

- 《悉达多》作者：赫尔曼·黑塞，译者：姜乙
  - 在朋友圈强烈推荐过。仿佛自己也跟着悉达多经历了一生，会想“我也这样想过”，“我现在在哪个阶段了？”……
  - “所有这些本能、简单、愚蠢，却极为强烈鲜活的欲望不再幼稚。他看到人们为欲望而活，因欲望不断创造、出行、征战，不断受难。他爱他们。他在他们的每种激情、每种作为中看到生命、生机，看到坚不可摧之物和梵天。他在他们盲目的忠诚、盲目的强悍和坚韧中看到可爱和可敬之处。世人和学者、思想者相比应有尽有，除了唯一微不足道的东西：自觉。对生命整体的自觉思考。”
- 《论语别裁》作者：南怀瑾
  - 在朋友圈也强烈推荐过。高人加好书的结合。如果我有孩子，我会先看完这本书。
  - 我还想过一个问题，父母应该教给孩子什么？我想，孩子作为一张白纸来到这个世界，应该让孩子学会与自然相处，懂得敬畏与共生；学会与他人相处，学会尊重与协作；学会与自我相处，建立清晰的认知与丰沛的内心。
  - 不管有没有孩子，都推荐看，都能从中获得启发。
- 《被讨厌的勇气》作者：岸见一郎
  - 几年前看的，不记得具体内容了。但看完后有一个感受：与他人相处，首先问心无愧，其次让对方舒服，做到第一点就行了，第二点做不到也没关系。
- 《我就是你啊：走进他人内心的 7 项修炼》作者：皮埃尔·佩利西耶
  - 和《高效能人士的七个习惯》中的双赢思维、知彼解己、综合统效有异曲同工之妙，但更为形象生动的描述了冲突是怎样产生的，如何避免冲突。
  - 很新颖的讲述方式，影响了我的写作方式。
- 哲学：马克思主义哲学等
  - 曾经我也是觉得思想政治课很无聊，很难理解的人，文科中我喜欢历史，地理也还行，但对政治课提不起兴趣，虽然我理科也一般，但还是选了理科。
  - 没想到多年后，我会觉得哲学也很有意思。有一次周会分享，被问到怎么个有意思法，我说不上来，我说“感觉很奇妙”。后来，我发现“哲学是对具体科学知识的概括和总结”这句话，就能解释清楚。
  - 当把一件事抽象出来、提炼出规律，会发现这个结论可以跨越领域，迁移到各种情境。这是之前觉得很奇妙的地方。
  - 而且学习哲学，会有一通则通的感觉，就像有了通用的通关秘籍。
  - 即使不看哲学书（本人也没有看过完整的纯哲学著作），也可以在日常中尝试总结概括，一段时间后，会发现表达、理解能力都得到了提升。

## 补充说明

上次发文《给北京遇到的伙伴们》末尾中，我说：

> “愿我们都平安健康，眉头舒展，笑口常开。有挚爱的人相伴，为热爱的事奋斗，与志同道合的人共事！”

我觉得幸福的一种外在表现是，眉头舒展。

什么时候会眉头舒展？

- 首先是身体不难受，健康、温饱、呼吸顺畅、睡眠充足、身体舒展；
- 其次是心理不难受，有挚爱的人（可以是家人、伴侣、朋友等）或其他生物（比如，宠物）或非生物（比如，二次元、一个物品）陪伴，做自己想做的事，内心平静没有负面情绪（恐惧、愤怒、焦虑、紧张、悲伤等）。

当满足这些条件时，应该就是幸福的。感恩自己拥有这么多。
